## 行为准则

参与本项目时，请保持尊重和专业的态度。我们欢迎各种形式的贡献，包括 bug 报告、功能请求、代码贡献和文档改进。

## 如何贡献

### 1. 报告 Bug

如果您发现了 bug，请通过 GitHub Issues 提交报告。在提交前，请先查看是否已有类似的 issue。

提交 bug 报告时，请包含：
- 详细的问题描述
- 复现步骤
- 预期行为和实际行为
- 环境信息（IDA Pro 版本、操作系统、Python 版本等）
- 如果可能，提供截图或错误日志

### 2. 提交功能请求

如果您有新功能的想法，可以通过 GitHub Issues 提交功能请求。请描述：
- 您想要的功能
- 这个功能解决的问题
- 可能的实现方式（如果您有想法）

### 3. 代码贡献

#### 准备工作

1. Fork 本仓库
2. 克隆您的 fork 到本地：
   ```bash
   git clone https://github.com/YOUR_USERNAME/idapromcp_333.git
   cd idapromcp_333
   ```
3. 创建一个新分支：
   ```bash
   git checkout -b feature-or-bugfix-branch
   ```

#### 开发流程

1. 安装开发依赖：
   ```bash
   pip install -e .
   ```

2. 进行代码修改

3. 测试您的修改

4. 提交您的更改：
   ```bash
   git add .
   git commit -m "描述您的更改"
   ```

5. 推送到您的 fork：
   ```bash
   git push origin feature-or-bugfix-branch
   ```

6. 创建 Pull Request

### 4. 文档贡献

文档改进也是非常宝贵的贡献。您可以通过修改 README.md 或添加更详细的使用示例来改进文档。

## 代码风格

请遵循 Python 的 PEP 8 风格指南。确保您的代码有适当的注释和文档字符串。

## 提交信息规范

请使用清晰、描述性的提交信息。好的提交信息应该清楚地说明您做了什么以及为什么这样做。
